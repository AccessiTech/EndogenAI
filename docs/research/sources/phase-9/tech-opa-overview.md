general-purpose policy engine that unifies policy enforcement across the stack.
OPA provides a high-level declarative language that lets you specify policy as
code and simple APIs to offload policy decision-making from your software. You
can use OPA to enforce policies in microservices, Kubernetes, CI/CD pipelines,
API gateways, and more.</p>
<p>OPA is proud to be a graduated
<a href="https://www.cncf.io/announcements/2021/02/04/cloud-native-computing-foundation-announces-open-policy-agent-graduation/" target="_blank" rel="noopener noreferrer" class="">Cloud Native Computing Foundation (CNCF)</a>
project.</p>
<p>This page covers core concepts in OPA&#x27;s policy language
(<a class="" href="/docs/policy-language">Rego</a>) as well as how to download and run OPA.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="what-is-opa">What is OPA?<a href="#what-is-opa" class="hash-link" aria-label="Direct link to What is OPA?" title="Direct link to What is OPA?" translate="no">​</a></h2>
<p>OPA <a class="" href="/docs/philosophy#policy-decoupling">decouples</a> policy decision-making from policy
enforcement. When your software needs to make policy decisions it <strong>queries</strong>
OPA and supplies structured data (e.g., JSON) as input. OPA accepts arbitrary
structured data as input.</p>
<!-- -->
<!-- -->
<p>OPA generates policy decisions by evaluating the query input against
policies and data. OPA and Rego are domain-agnostic so you can describe almost
any kind of invariant in your policies. For example:</p>
<ul>
<li class="">Which users can access which resources.</li>
<li class="">Which subnets egress traffic is allowed to.</li>
<li class="">Which clusters a workload must be deployed to.</li>
<li class="">Which registries binaries can be downloaded from.</li>
<li class="">Which OS capabilities a container can execute with.</li>
<li class="">Which times of day the system can be accessed at.</li>
</ul>
<p>Policy decisions are not limited to simple yes/no or allow/deny answers. Like
query inputs, your policies can generate arbitrary structured data as output.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="basic-syntax">Basic Syntax<a href="#basic-syntax" class="hash-link" aria-label="Direct link to Basic Syntax" title="Direct link to Basic Syntax" translate="no">​</a></h3>
<p>To implement our security policy, we first need to access and examine the
infrastructure data. When OPA evaluates policies, it binds data provided in the

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="2-try-opa-eval">2. Try <code>opa eval</code><a href="#2-try-opa-eval" class="hash-link" aria-label="Direct link to 2-try-opa-eval" title="Direct link to 2-try-opa-eval" translate="no">​</a></h3>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="3-try-opa-run-interactive">3. Try <code>opa run</code> (interactive)<a href="#3-try-opa-run-interactive" class="hash-link" aria-label="Direct link to 3-try-opa-run-interactive" title="Direct link to 3-try-opa-run-interactive" translate="no">​</a></h3>
<p>OPA includes an interactive shell or REPL (Read-Eval-Print-Loop) accessible via
the <a class="" href="/docs/cli#run"><code>opa run</code> sub-command</a>.
You can use the REPL to experiment with policies and prototype new ones.</p>
<p>To start the REPL just:</p>
<div class="language-bash codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-bash codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token plain">./opa run</span><br></span></code></pre></div></div>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="4-try-opa-run-server">4. Try <code>opa run</code> (server)<a href="#4-try-opa-run-server" class="hash-link" aria-label="Direct link to 4-try-opa-run-server" title="Direct link to 4-try-opa-run-server" translate="no">​</a></h3>
<p>To integrate with OPA you can run it as a server and execute queries over HTTP.
