It consists of two major parts:</p><ul><li><p>A dev server that provides <a href="./features">rich feature enhancements</a> over <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Guide/Modules" target="_blank" rel="noreferrer">native ES modules</a>, for example extremely fast <a href="./features#hot-module-replacement">Hot Module Replacement (HMR)</a>.</p></li><li><p>A build command that bundles your code with <a href="https://rollupjs.org" target="_blank" rel="noreferrer">Rollup</a>, pre-configured to output highly optimized static assets for production.</p></li></ul><p>Vite is opinionated and comes with sensible defaults out of the box. Read about what&#39;s possible in the <a href="./features">Features Guide</a>. Support for frameworks or integration with other tools is possible through <a href="./using-plugins">Plugins</a>. The <a href="./../config/">Config Section</a> explains how to adapt Vite to your project if needed.</p><p>Vite is also highly extensible via its <a href="./api-plugin">Plugin API</a> and <a href="./api-javascript">JavaScript API</a> with full typing support.</p><p>You can learn more about the rationale behind the project in the <a href="./why">Why Vite</a> section.</p><div class="scrimba" data-v-d3e6fdfd><span class="play-button" data-v-d3e6fdfd><span class="play-icon" data-v-d3e6fdfd></span></span><a href="https://scrimba.com/intro-to-vite-c03p6pbbdq?via=vite" target="_blank" rel="sponsored noopener" title="Free Vite Course on Scrimba" data-v-d3e6fdfd><!--[-->Learn Vite through interactive tutorials on Scrimba<!--]--></a></div><h2 id="browser-support" tabindex="-1">Browser Support <a class="header-anchor" href="#browser-support" aria-label="Permalink to “Browser Support”">​</a></h2><p>During development, Vite assumes that a modern browser is used. This means the browser supports most of the latest JavaScript and CSS features. For that reason, Vite sets <a href="https://esbuild.github.io/api/#target" target="_blank" rel="noreferrer"><code>esnext</code> as the transform target</a>. This prevents syntax lowering, letting Vite serve modules as close as possible to the original source code. Vite injects some runtime code to make the development server work. These code use features included in <a href="https://web-platform-dx.github.io/web-features/" target="_blank" rel="noreferrer">Baseline</a> Newly Available at the time of each major release (2025-05-01 for this major).</p><p>For production builds, Vite by default targets <a href="https://web-platform-dx.github.io/web-features/" target="_blank" rel="noreferrer">Baseline</a> Widely Available browsers. These are browsers that were released at least 2.5 years ago. The target can be lowered via configuration. Additionally, legacy browsers can be supported via the official <a href="https://github.com/vitejs/vite/tree/main/packages/plugin-legacy" target="_blank" rel="noreferrer">@vitejs/plugin-legacy</a>. See the <a href="./build">Building for Production</a> section for more details.</p><h2 id="trying-vite-online" tabindex="-1">Trying Vite Online <a class="header-anchor" href="#trying-vite-online" aria-label="Permalink to “Trying Vite Online”">​</a></h2><p>You can try Vite online on <a href="https://vite.new/" target="_blank" rel="noreferrer">StackBlitz</a>. tabindex="-1"><code>index.html</code> and Project Root <a class="header-anchor" href="#index-html-and-project-root" aria-label="Permalink to “index.html and Project Root”">​</a></h2><p>One thing you may have noticed is that in a Vite project, <code>index.html</code> is front-and-central instead of being tucked away inside <code>public</code>. This is intentional: during development Vite is a server, and <code>index.html</code> is the entry point to your application.</p><p>Vite treats <code>index.html</code> as source code and part of the module graph. It resolves <code>&lt;script type=&quot;module&quot; src=&quot;...&quot;&gt;</code> that references your JavaScript source code. Even inline <code>&lt;script type=&quot;module&quot;&gt;</code> and CSS referenced via <code>&lt;link href&gt;</code> also enjoy Vite-specific features. In addition, URLs inside <code>index.html</code> are automatically rebased so there&#39;s no need for special <code>%PUBLIC_URL%</code> placeholders.</p><p>Similar to static http servers, Vite has the concept of a &quot;root directory&quot; which your files are served from. You will see it referenced as <code>&lt;root&gt;</code> throughout the rest of the docs. Absolute URLs in your source code will be resolved using the project root as base, so you can write code as if you are working with a normal static file server (except way more powerful!). Vite is also capable of handling dependencies that resolve to out-of-root file system locations, which makes it usable even in a monorepo-based setup.</p><p>Vite also supports <a href="./build#multi-page-app">multi-page apps</a> with multiple <code>.html</code> entry points.</p><h4 id="specifying-alternative-root" tabindex="-1">Specifying Alternative Root <a class="header-anchor" href="#specifying-alternative-root" aria-label="Permalink to “Specifying Alternative Root”">​</a></h4><p>Running <code>vite</code> starts the dev server using the current working directory as root. You can specify an alternative root with <code>vite serve some/sub/dir</code>.
