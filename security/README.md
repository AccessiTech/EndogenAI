# security/

This directory contains OPA policy files, generated policy data, certificates, and security review documentation for EndogenAI.

## Structure

| Path | Purpose |
|------|---------|
| `policies/` | OPA Rego policy rules (module capabilities, inter-module comms) |
| `data/modules.json` | Auto-generated OPA data from `agent-card.json` files — re-run `scripts/gen_opa_data.py` after changes |
| `tests/` | OPA unit tests — run with `opa test security/policies/` |
| `review/` | Phase 9 inter-module security review document |
| `certs/` | **Gitignored** — self-signed mTLS CA and per-module certs (generated by `scripts/gen_certs.sh`) |

## Usage

### Start OPA server

```bash
docker compose --profile security up -d opa-security
curl http://localhost:8181/health
```

### Regenerate OPA data

```bash
uv run python scripts/gen_opa_data.py
# Outputs: security/data/modules.json (14 modules)
```

### Run policy tests

```bash
opa test security/policies/ -v
```

### Generate mTLS certificates

```bash
bash scripts/gen_certs.sh
# Output: security/certs/{ca,<module>}.{crt,key}
```

## Security profile (Docker Compose)

The `security` profile adds the OPA server. The `docker-compose.override.yml` disables gVisor for local macOS dev.

```bash
docker compose --profile security up -d
```

## Policy design

- **Default deny**: all capabilities and A2A calls are denied unless explicitly allowed
- **Endogenous data**: `security/data/modules.json` is derived directly from `agent-card.json` files, ensuring policy is grounded in system knowledge
- **OPA_ENFORCE=false**: audit mode during development; set to `true` in production

See `docs/guides/security.md` for full security architecture documentation.
